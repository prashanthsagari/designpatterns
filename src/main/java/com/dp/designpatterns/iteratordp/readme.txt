OO Pattern:
**********
The Iterator Pattern : provides a way to access the elements of an aggregate object sequentially without exposing its underlying representation.
Composite Pattern: allows you to compose objects into tree structures to represent part-whole bierarchies. Composite lets clients treat individual objects and compositions of 
objects uniformly.
